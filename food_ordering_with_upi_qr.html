Project Overview

Objective: To test the functionality and usability of a demo online food ordering application. The goal is to ensure core flows work correctly (login, menu browsing, cart operations, checkout, payment selection) and the interface remains easy to understand on mobile and desktop.

Scope: Login, Search, Category browsing, Add/Remove items, Quantity update, Checkout, Payment selection, Order confirmation.

Test Plan

Test Objectives
  • Verify login form validations and user flow.
  • Validate menu browsing, filtering, and searching.
  • Ensure cart operations (add, remove, update quantity) work correctly.
  • Verify checkout and order confirmation behavior.
  • Evaluate usability on mobile-first layout.
Test Environment
  • Browser: Chrome / Edge (latest)
  • Device: Mobile + Desktop responsive view
  • OS: Windows / Android
  • Storage: LocalStorage used for cart data
Test Types
  • Functional Testing: Verify expected behavior of each feature.
  • Usability Testing: Check ease of use, clarity, and mobile friendliness.
Entry / Exit Criteria
  • Entry: UI pages available, dummy data loaded, cart storage enabled.
  • Exit: All critical test cases executed, major bugs documented, summary report prepared.

Usability Testing Feedback

User Feedback Points
  • Search bar is easy to find on the home screen.
  • Category cards are visually clear and help quick navigation.
  • Cart page summary improves confidence before checkout.
UI Improvement Suggestions
  • Buttons should be larger on mobile for better accessibility.
  • Checkout steps should be clearer (Address ‚Üí Payment ‚Üí Confirm).
  • Add a small toast message when item is added to cart.
  • Show veg/non-veg indicator icon on menu cards.
  • Reduce number of navbar items on very small screens.
